# CLAUDE.md — Frontend

This file provides guidance to Claude Code when working in `frontend/`.
Read the root `../CLAUDE.md` first for shared context (project overview, Sleeve model, monorepo layout).

## Tech Stack

React 18 + TypeScript + Vite · Tailwind · TanStack Query · React Router v6 · Recharts · Axios · openapi-typescript

## Commands

Run from `frontend/`.

```bash
npm run dev           # dev server at http://localhost:5173
npm run build         # tsc + vite build
npm run lint          # eslint
npm run gen:api       # regenerate src/api/schema.d.ts from ../openapi.yaml
```

`npm run gen:api` must be re-run after any backend `openapi.yaml` change. `frontend/openapi.yaml` is a generated artifact — never edit it by hand.

## Architecture

### Data layer

```
src/api/
  client.ts           # Axios instance — base URL from VITE_API_BASE_URL, auth interceptor seam
  schema.d.ts         # generated by openapi-typescript — do not hand-edit
  endpoints/*.ts      # one file per resource (clients, accounts, sleeves, strategies, trades, admin)
src/hooks/            # one React Query hook per resource (useClients, useSleeves, useAdmin, ...)
```

Every endpoint has a thin typed wrapper in `src/api/endpoints/` and a matching hook in `src/hooks/`. Hooks are the only place TanStack Query is imported — feature components call hooks, not `apiClient` directly.

### State

Two React contexts drive all data-fetching:
- `ScopeContext` — the currently selected `{ clientId?, accountId?, sleeveId? }`. Changing scope invalidates performance queries.
- `DateRangeContext` — `{ startDate, endDate }` fed as query params to all performance endpoints.

Both live in `src/context/`. Every performance screen reads from these; never accept scope or dates as local state.

### Routing

All app routes nest under `<AppShell>` at `/app/*`. Route params:
- `/app/clients/:clientId`
- `/app/clients/:clientId/accounts/:accountId`
- `/app/clients/:clientId/accounts/:accountId/sleeves/:sleeveId`

Fast-follower routes (`/app/portfolio`, `/app/risk`, `/app/grow`) exist but render `<ComingSoon>`.

### Shared components (`src/components/`)

| Component | Purpose |
|---|---|
| `AppShell` | Left sidebar + scope switcher + date picker; wraps all app routes |
| `StatCard` | Single metric display (label + value + optional delta) |
| `PerformanceChart` | Recharts line chart for `timeseries: [{date, value}]` |
| `MonthlyReturnsHeatmap` | Year × month grid from `/performance/monthly`; gain/clay ramp |
| `LevelBreakdown` | Sortable ranked table of `children[]` with links to drill-down |
| `DataTable` | Generic sortable table; sticky header; right-aligned mono numbers |
| `EmptyState` / `ErrorState` | Consistent empty and error treatments; every screen uses these |
| `Skeleton` | Loading placeholders; every data-fetching screen shows skeletons |
| `ComingSoon` | Placeholder for fast-follower modules (Portfolio, Risk, Grow) |

### Features (`src/features/`)

Each feature directory contains its page component(s). Implemented:
- `performance/PerformancePage` — hero MWR/TWR, timeseries, heatmap, level breakdown
- `clients/ClientsPage`, `clients/ClientDetailPage`
- `accounts/AccountDetailPage`
- `sleeves/SleeveDetailPage`
- `strategies/StrategiesPage`
- `imports/ImportsPage` — two-phase VALIDATE→IMPORT flow, sync logs, sheet sync configs

Stubs (render `<ComingSoon>`): `PortfolioPage`, `RiskPage`, `GrowPage`, `SettingsPage`.

### Formatters (`src/lib/formatters.ts`)

Central formatters for currency, percent, and dates — use these for every number that reaches the screen. All numbers render in `font-mono` with `font-feature-settings: "tnum"` (tabular figures), right-aligned in tables.

## Design Tokens

Custom tokens are in `tailwind.config.ts`. Brand rules that must hold:
- **Losses are clay, never red** — use `text-loss` / `bg-loss` (`#C2703D`), never `text-red-*`
- **Gold is the only accent** — one primary CTA per screen, highlighted figures
- **All numbers: `font-mono` + tabular figures** (`font-feature-settings: "tnum"`)
- **Gains: `text-gain`** (`#2E8B6B`)

```
summit-ink #14201C   pine-deep #1E3A2F   pine #2F5D4A
mist #EAF0EC         paper #FBFCFB       stone #8A968F
gain #2E8B6B         loss #C2703D        gold #C9A24B
```

## Env Setup

Copy `.env.example` to `.env`. Required var: `VITE_API_BASE_URL=http://localhost:8000`.

## Fast-Follower Modules (do not wire to endpoints)

Portfolio, Risk, Grow, benchmark comparison, and auth have no backend yet. Render `<ComingSoon>` and record gaps in `BACKEND_GAPS.md`. The API also lacks a list-all-clients endpoint — known clients are tracked locally until one lands.

## What "Done" Looks Like

- Component reads from a hook, not raw `apiClient`
- `isLoading` → `<Skeleton>`, `isError` → `<ErrorState>` with retry, empty → `<EmptyState>`
- Numbers formatted via `src/lib/formatters.ts`, gains green, losses clay
- No mock data in committed paths once the real endpoint works
- `openapi.yaml` updated if a new endpoint was added
