## Tech Stack

React 18 + TypeScript + Vite ¬∑ Tailwind ¬∑ TanStack Query ¬∑ React Router v6 ¬∑ Recharts ¬∑ Axios ¬∑ openapi-typescript

## Commands

Run from `frontend/`.

```bash
npm run dev           # dev server at http://localhost:5173
npm run build         # tsc + vite build
npm run lint          # eslint
npm run gen:api       # regenerate src/api/schema.d.ts from ../openapi.yaml
```

`npm run gen:api` must be re-run after any backend `openapi.yaml` change. `frontend/openapi.yaml` is a generated artifact ‚Äî never edit it by hand.

## Architecture

### Data layer

```
src/api/
  client.ts           # Axios instance ‚Äî base URL from VITE_API_BASE_URL, auth interceptor seam
  schema.d.ts         # generated by openapi-typescript ‚Äî do not hand-edit
  endpoints/*.ts      # one file per resource (clients, accounts, sleeves, strategies, trades, admin)
src/hooks/            # one React Query hook per resource (useClients, useSleeves, useAdmin, ...)
```

Every endpoint has a thin typed wrapper in `src/api/endpoints/` and a matching hook in `src/hooks/`. Hooks are the only place TanStack Query is imported ‚Äî feature components call hooks, not `apiClient` directly.

### State

Two React contexts drive all data-fetching:
- `ScopeContext` ‚Äî the currently selected `{ clientId?, accountId?, sleeveId? }`. Changing scope invalidates performance queries.
- `DateRangeContext` ‚Äî `{ startDate, endDate }` fed as query params to all performance endpoints.

Both live in `src/context/`. Every performance screen reads from these; never accept scope or dates as local state.

### Routing

All app routes nest under `<AppShell>` at `/app/*`. Route params:
- `/app/clients/:clientId`
- `/app/clients/:clientId/accounts/:accountId`
- `/app/clients/:clientId/accounts/:accountId/sleeves/:sleeveId`

Fast-follower routes (`/app/portfolio`, `/app/risk`, `/app/grow`) exist but render `<ComingSoon>`.

### Data layer conventions

- **Field names must match the backend exactly.** Key ones: `mwr_pct`, `twr_pct`, `total_market_value`, `total_unrealized_gain` (on `PerformanceSummary`); `matched_quantity`, `entry_price`, `exit_price`, `opened_at`, `closed_at` (on `ClosedPosition`); `months` not `monthly_returns` (on `MonthlyReturnsResponse`).
- **Percentage units**: `mwr_pct`/`twr_pct`/`twr_cumul`/`mwr_cumul` are decimal fractions (0.05 = 5%); `unrealized_gain_pct`/`realized_gain_pct`/`return_pct` are whole-number percentages (7.34 = 7.34%). Pass decimals to `formatPercent`, whole numbers to display with `%` directly.
- **All IDs are UUIDs** (string, 36 chars) ‚Äî never assume numeric.
- **Timestamps are ISO-8601 strings** in API responses.

### Navigation to PerformancePage

When navigating to `/app/performance` from a detail page, always wrap both the scope setter and `navigate()` in `flushSync()` to avoid React batching the state update after the page mounts:

```tsx
import { flushSync } from 'react-dom'
flushSync(() => setClient(id, name))
navigate('/app/performance')
```

This pattern is in `ClientsPage`, `ClientDetailPage`, `AccountDetailPage`, and `SleeveDetailPage`. Do not remove it ‚Äî without it the performance page mounts before scope is set and renders blank.

### Performance chart date filtering

The backend always returns full-history `timeseries` regardless of `start_date`/`end_date` params. Filter on the client side before passing to `PerformanceChart` and `ReturnsSeriesChart`:

```tsx
data={perf.timeseries.filter(pt => {
  if (startDate && pt.date < startDate) return false
  if (endDate && pt.date > endDate) return false
  return true
})}
```

The quick-filter presets (`applyPreset`) set `DateRangeContext`; the chart section then re-filters from the already-fetched full series.

### Formatters (`src/lib/formatters.ts`)

Central formatters for currency, percent, and dates ‚Äî use these for every number that reaches the screen. All numbers render in `font-mono` with `font-feature-settings: "tnum"` (tabular figures), right-aligned in tables.

## Design Tokens

Custom tokens are in `tailwind.config.ts`. Brand rules that must hold:
- **Losses are clay, never red** ‚Äî use `text-loss` / `bg-loss` (`#C2703D`), never `text-red-*`
- **Gold is the only accent** ‚Äî one primary CTA per screen, highlighted figures
- **All numbers: `font-mono` + tabular figures** (`font-feature-settings: "tnum"`)
- **Gains: `text-gain`** (`#2E8B6B`)

```
summit-ink #14201C   pine-deep #1E3A2F   pine #2F5D4A
mist #EAF0EC         paper #FBFCFB       stone #8A968F
gain #2E8B6B         loss #C2703D        gold #C9A24B
```

Portfolio, Risk, Grow, benchmark comparison, and auth have no backend yet. Render `<ComingSoon>` and record gaps in `BACKEND_GAPS.md`. The API also lacks a list-all-clients endpoint ‚Äî known clients are tracked locally until one lands.

## What "Done" Looks Like

- Component reads from a hook, not raw `apiClient`
- `isLoading` ‚Üí `<Skeleton>`, `isError` ‚Üí `<ErrorState>` with retry, empty ‚Üí `<EmptyState>`
- Numbers formatted via `src/lib/formatters.ts`, gains green, losses clay
- No mock data in committed paths once the real endpoint works
- `openapi.yaml` updated if a new endpoint was added
